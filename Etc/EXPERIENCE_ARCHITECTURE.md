# CareerOS Experience Architecture

This document defines the emotional and operational experience flows across the CareerOS ecosystem. CareerOS is an interconnected career identity platform; every interaction must feel intelligent, cohesive, and intentional.

## Core UX Philosophy
- **Alive but Calm:** The system should feel like it is continuously working for the user without generating anxiety.
- **Workflow First:** Users are not "browsing pages." They are moving through pipelines and intelligent workflows.
- **Contextual Intelligence:** The AI does not sit in a corner. It adapts its UI (persistent right panel + inline intelligence) based on the user's current environment.

---

## 1. Discovery Experience (The Retention Engine)
**Emotional State:** *Exciting, Confidence-Building, Intelligent.*
**Operational State:** High-density scanning, quick decision-making.

- **Feel:** Feels like looking at a Bloomberg terminal of career opportunities, but perfectly tailored.
- **Interactions:** Fast keyboard navigation (j/k to move between opportunities).
- **AI Integration:** Contextual fit explanations appear instantly. Ghost job signals warn users transparently.
- **Noise Reduction:** Discovery must reduce noise aggressively. It should feel curated, not like infinite-scroll chaos.

## 2. Tracker Experience (The Operations Engine)
**Emotional State:** *Structured, Calm, Operational.*
**Operational State:** Pipeline management, drag-and-drop continuity.

- **Feel:** Like Airtable combined with Linear. Zero lag. Crisp borders.
- **Interactions:** Fluid drag-and-drop. Every workflow transition should feel trustworthy and deterministic. Users must TRUST the workflow system.
- **AI Integration:** Proactive reminders ("Follow up with Stripe recruiter today").

## 3. ATS Intelligence & Resume Lab
**Emotional State:** *Analytical, Empowering, Educational.*
**Operational State:** Deep focus, iterative optimization.

- **Feel:** Like a Notion document infused with a compiler's intelligence.
- **Interactions:** Inline text suggestions. Heatmap highlighting over resume keywords.
- **AI Integration:** The ATS Agent provides real-time scoring adjustments as the user types. The system should teach users how hiring systems work, acting as an educational infrastructure moat.

## 4. Marketplace & Profile Push
**Emotional State:** *Alive, Network-Driven, Passive Confidence.*
**Operational State:** Discoverability tuning, stealth configuration.

- **Feel:** Like an exclusive talent network graph.
- **Interactions:** Satisfying toggle switches for "Stealth Mode" or "Active Seeking." Interactions should be high-signal and intentional (no spammy network effects or social-media energy).
- **AI Integration:** "Your profile was viewed by 3 recruiters today. Your discoverability score is 92%."

## 5. Global Command System (The Universal Navigator)
**Emotional State:** *Omnipotent, Instant.*
**Operational State:** Search, Action, Intelligence all in one keystroke.

- **Feel:** Raycast.
- **Interactions:** `Cmd+K` anywhere. Immediate fuzzy search results across jobs, recruiters, docs, candidates, and AI workflows. This is the universal orchestration infrastructure.

## 6. Public Landing & Marketing Ecosystem
**Emotional State:** *Cinematic, Futuristic, Trust-Inducing.*
**Operational State:** Narrative scrolling, interactive demos.

- **Feel:** Vercel meets Apple. Highly polished, subtle 3D effects or complex GSAP staggered animations to reveal the ecosystem depth. (Cinematic outside, operational inside).
- **Interactions:** Scroll-based reveals of the Discovery -> ATS -> Tracker pipeline.

---

## 7. System Continuity Principles (CRITICAL)

Because CareerOS is ecosystem software, not a collection of isolated pages, continuity is paramount:
- Every system should feel interconnected.
- Resume changes should affect discovery instantly.
- Tracker actions should influence recommendations.
- Marketplace visibility should adapt dynamically based on pipeline health.
- AI memory should persist across all workflows.
- Recommendations should evolve continuously.
- The user should never feel context resets.

---

## 8. Trust Architecture

Users are trusting CareerOS with their careers, resumes, and recruiter visibility. This is HIGH trust software.
- **Explainability:** AI decisions must be explainable.
- **Transparency:** Ghost scores should be fully transparent.
- **Controllability:** Automation should always feel controllable.
- **Clarity:** Users should understand why specific recommendations appear.
- **Non-Manipulative:** The system should never use dark patterns or manipulative UX.
- **User-Controlled:** Recruiter visibility should always remain strictly under the user's control.
